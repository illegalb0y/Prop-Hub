## 1. Objective
Design and implement a **beautiful, minimalistic admin panel UI** that enables internal users to:
* Monitor **product analytics and user behavior**
* Manage **core platform data** (projects, developers, banks)
* Navigate efficiently via a **clear sidebar-based information architecture**

* modern
* calm
* premium
* trustworthy
Comparable to high-end SaaS tools such as **Attio**, Linear, Stripe, Vercel.

## 4. Navigation Structure & Pages
### 4.1 Product Analytics
#### 4.1.1 Users Page
**Purpose**
Provide visibility into user activity and security-related metadata.
**Content**
* User list table
* Each row includes:
  * User ID
  * Role
  * Active / inactive status
  * Last session time
  * IP address
  * Device / browser (optional)
* Row click opens **User Details Drawer / Page**
**Actions**
* View user sessions
* Change role (if permitted)
* Deactivate / activate user
**UX Notes**
* Use subtle row hover
* No heavy borders
* Inline status indicators (dots, pills)
---
#### 4.1.2 Dashboard Page
**Purpose**
High-level product analytics overview.
**Metrics (MVP)**
* Daily / Monthly Active Users (DAU / MAU)
* Sessions
* Retention
* Churn
* Conversion rate
**Components**
* KPI cards (top row)
* Line charts for trends
* Filters:
  * Date range
  * Granularity (daily / weekly / monthly)
**UX Notes**
* Avoid clutter
* Prioritize readability
* Charts should feel lightweight and calm
---
### 4.2 Data Section
Each data entity has its **own CRUD page**, sharing a consistent layout and interaction pattern.
---
#### 4.2.1 Projects Page
**Purpose**
Manage real estate projects displayed in the product.
**Features**
* Table view with:
  * Name
  * Status
  * Developer
  * Created date
* Actions:
  * Create project
  * Edit project
  * Delete project
* Search + basic filters
---
#### 4.2.2 Developers Page
**Purpose**
Manage developers linked to projects.
**Features**
* CRUD functionality
* Status toggle (active / inactive)
* Relationship visibility (linked projects count)
---
#### 4.2.3 Banks Page
**Purpose**
Manage banks associated with mortgage / financing options.
**Features**
* CRUD functionality
* Simple form-based creation and editing

## 5. Visual & Design Requirements
### Design Style
* Minimalistic
* Premium SaaS look
* No heavy borders
* Generous whitespace
### Typography
* Modern sans-serif
* Clear hierarchy
* Comfortable reading size
Examples:
* Inter
* SF Pro
* IBM Plex Sans
---
### Color Palette
* Neutral base (white / soft gray)
* One primary accent color
* Subtle gradients or gloss effects (very light)
---
### UI Elements
* Rounded corners (medium)
* Soft shadows
* Smooth hover & focus states
* Drawer-based forms preferred over modals
